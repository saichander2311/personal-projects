# What is SDLC? 🚀

**SDLC (Software Development Life Cycle)** is a **step-by-step process used to design, develop, test, and deploy software**.

Think of it like building a house:
- You don't start with construction immediately
- First, you plan, design, build, inspect, and then hand it over

Software development works in the same way.

## Why is SDLC Important?

Without SDLC, software development becomes chaotic.

**SDLC helps in:**
- Avoiding confusion and mistakes
- Managing time and cost effectively
- Improving software quality
- Ensuring customer satisfaction
- Making teamwork easier

**In short:** SDLC brings **discipline and structure** to development.

## Phases of SDLC

SDLC consists of 7 main phases. Each has a specific purpose.

### 1. Requirement Gathering & Analysis
**First and most important phase.**

Team:
- Talks to clients/stakeholders
- Understands software needs
- Collects all requirements

**Example:** Food delivery app needs user login, item listing, payment system.

> *Poor requirements = project failure*

### 2. Planning
Creates project roadmap.

Decides:
- Timeline
- Budget
- Resources
- Risks

**Question answered:** "How are we going to build this?"

### 3. Design
Creates system structure.

Includes:
- Architecture design
- Database design
- UI/UX design

**Types:**
- **High-Level Design (HLD)** — overall system
- **Low-Level Design (LLD)** — component details

### 4. Development (Coding)
Developers write code.

- Build modules
- Implement features
- Follow coding standards

**Longest phase.**

### 5. Testing
Ensures software quality.

**Types:**
- Unit Testing
- Integration Testing
- System Testing
- User Acceptance Testing (UAT)

### 6. Deployment
Release to users.

**Options:**
- Full release
- Gradual/phased release

**Example:** App Store launch.

### 7. Maintenance
Post-release support.

Includes:
- Bug fixes
- New features
- Performance improvements

## SDLC Models

| Model | Characteristics | Best For |
|-------|-----------------|----------|
| **Waterfall** | Linear, sequential | Simple, fixed projects |
| **Agile** | Iterative, flexible | Dynamic requirements |
| **Spiral** | Risk-focused, iterative | Complex, high-risk projects |

## Real-Life Example: Online Shopping App

1. **Requirement** — Browse, cart, pay
2. **Planning** — Timeline & team
3. **Design** — App layout & database
4. **Development** — Code features
5. **Testing** — Login, payments, bugs
6. **Deployment** — Launch app
7. **Maintenance** — Fix issues, add features

## Conclusion

**SDLC ensures software is:**
- Well-planned
- Properly built
- Thoroughly tested
- Successfully delivered
